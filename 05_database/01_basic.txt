
DATABASE BASICS


1. WHAT IS DATA?
Data = Raw facts or information.

Simple meaning:
Data is unprocessed and has no meaning until it is organized.

Examples:
- Vikash
- 22
- Delhi
- 85

These are just values → no context.

When we combine them:
Vikash, 22, Delhi → Now it becomes meaningful → Information


------------------------------------------------------------


2. WHAT IS DATABASE?
---------------------
Database = Organized collection of data.

Simple meaning:
Data stored in a structured way so that it can be easily:
- Stored
- Retrieved
- Updated
- Deleted

Example (Student Table):

Name     Age   Course
----------------------
Vikash   22    BTech
Aman     21    BCA

This structured data = Database


------------------------------------------------------------


3. WHAT IS DBMS?
----------------
DBMS = Database Management System

Simple meaning:
Software that helps us manage databases.

Examples:
- MySQL
- MongoDB
- PostgreSQL

What DBMS does:
- Stores data
- Retrieves data
- Updates data
- Deletes data
- Provides security
- Handles multiple users

Real-life example:
When you login to Instagram or Amazon,
DBMS is working in the backend.


------------------------------------------------------------


4. FILE-BASED SYSTEM (OLD METHOD)
---------------------------------
Before DBMS, data was stored in files.

Example:
- students.txt
- employees.txt
- orders.txt

Each file stores data separately.

Problem:
No connection between files.


------------------------------------------------------------


5. ADVANTAGES OF FILE-BASED SYSTEM
----------------------------------
- Simple to use
- Easy to create files
- No special software needed
- Low cost


------------------------------------------------------------


6. DISADVANTAGES OF FILE-BASED SYSTEM
--------------------------------------

1. Data Redundancy (Duplicate Data)
   Same data stored multiple times.

   Example:
   students.txt → Vikash
   fees.txt → Vikash

2. Data Inconsistency
   Data updated in one file but not in another.

3. Difficult Data Access
   No easy way to search or query data.

4. No Security
   Anyone can access files.

5. No Relationships
   Files are independent, no connection between them.

6. No Backup & Recovery
   If file is deleted → data lost permanently.


------------------------------------------------------------


7. WHY DBMS IS BETTER?
----------------------

Feature              File System      DBMS
--------------------------------------------
Data duplication     High             Low
Security             No               Yes
Relationships        No               Yes
Querying             Hard             Easy
Backup               No               Yes


------------------------------------------------------------


FINAL UNDERSTANDING
-------------------

Flow:
Data → Database → DBMS

- Data = Raw facts
- Database = Organized data
- DBMS = Tool to manage data
