1. WHAT IS CLASSIFICATION OF DATA MODELS?
Classification of data models means grouping different ways of organizing data in a database.

Explanation:
Different data models define different ways to store and connect data. Each model has its own structure, advantages, and use cases.

Main Types:
1. Hierarchical Model
2. Network Model
3. Relational Model
4. Object-Oriented Data Model


2. HIERARCHICAL DATA MODEL
--------------------------
Hierarchical model stores data in a tree structure with parent-child relationships.

Explanation:
In this model, data is organized like a tree where each child node has only one parent. Data is accessed from the root downwards.

Structure:
Root → Parent → Child

Example:
Company
  └── Department
        └── Employee

Real-life Example:
A folder system in a computer.

Advantages:
- Simple structure
- Fast access for hierarchical data

Disadvantages:
- Rigid structure
- No multiple parent relationships


------------------------------------------------------------


3. NETWORK DATA MODEL
---------------------
Network model stores data in a graph structure allowing multiple relationships.

Explanation:
Unlike hierarchical model, a child can have multiple parents. It supports many-to-many relationships and is more flexible.

Structure:
Node ↔ Node (multiple connections)

Example:
Student ↔ Course
A student can enroll in multiple courses, and a course can have multiple students.

Advantages:
- Flexible relationships
- Supports complex data structures

Disadvantages:
- Complex design
- Difficult to maintain


------------------------------------------------------------


4. RELATIONAL DATA MODEL
------------------------
Relational model stores data in tables with rows and columns.

Explanation:
This is the most widely used model. Data is stored in tables, and relationships are defined using keys (Primary Key and Foreign Key).

Structure:
Table (Rows + Columns)

Example:

Students Table:
ID   Name
1    Vikash

Courses Table:
CID  Course
101  BTech

Enrollment Table:
ID   CID
1    101

Advantages:
- Simple and easy to understand
- Powerful querying using SQL
- Flexible and scalable

Disadvantages:
- Complex joins can be slow
- Not ideal for unstructured data


------------------------------------------------------------


5. OBJECT-ORIENTED DATA MODEL
-----------------------------
Object-oriented model stores data in the form of objects similar to programming languages.

Explanation:
This model combines database concepts with object-oriented programming. Data is stored as objects with properties (attributes) and methods (functions).

Concepts:
- Object
- Class
- Inheritance
- Encapsulation

Example:

Class: Student
Attributes: name, age
Methods: getDetails()

Object:
Student1 = { name: Vikash, age: 22 }

Advantages:
- Matches real-world entities
- Easy integration with OOP languages like Java, C++

Disadvantages:
- Complex implementation
- Less widely used compared to relational model
