1. WHAT IS RDBMS?
RDBMS is a database system that stores data in the form of tables with relationships.

Explanation:
RDBMS organizes data into tables (relations) where each table consists of rows and columns. It allows linking tables using keys, making data structured and easy to manage.

Examples:
- MySQL
- PostgreSQL
- Oracle

Real-life Example:
Users table connected with Orders table in an e-commerce app.


------------------------------------------------------------


2. WHAT IS A TABLE?
A table is a collection of related data organized in rows and columns.

Explanation:
Each table represents a specific entity like users, products, or orders. Tables are the basic building blocks of RDBMS.

Example:

Users Table:
ID   Name   Age
1    Vikash 22


------------------------------------------------------------


3. WHAT IS A ROW (TUPLE)?
A row represents a single record in a table.

Explanation:
Each row contains complete information about one entity.

Example:
(1, Vikash, 22) → One user


------------------------------------------------------------


4. WHAT IS A COLUMN (ATTRIBUTE)?
A column represents a property or field of data.

Explanation:
Each column stores a specific type of data.

Example:
Name, Age, ID are columns in a table.


------------------------------------------------------------


5. WHAT IS A PRIMARY KEY?
A primary key uniquely identifies each row in a table.

Explanation:
- Must be unique
- Cannot be NULL

Example:
User ID in Users table

Why important:
Ensures no duplicate records.


------------------------------------------------------------


6. WHAT IS A FOREIGN KEY?
A foreign key is a column that creates a link between two tables.

Explanation:
It refers to the primary key of another table.

Example:

Users Table:
ID = 1

Orders Table:
user_id = 1 → connects to Users table

Why important:
Maintains relationships between tables.


------------------------------------------------------------


7. WHAT IS A RELATIONSHIP?
A relationship defines how two tables are connected.

Types:

1. One-to-One
   One user → One profile

2. One-to-Many
   One user → Many orders

3. Many-to-Many
   Many students ↔ Many courses

Explanation:
Relationships are implemented using foreign keys.




8. WHAT IS NORMALIZATION?
Normalization is the process of organizing data to reduce redundancy.

Explanation:
It divides data into multiple tables to avoid duplication and maintain consistency.

Example:
Instead of storing user info in every order,
store user in one table and reference it.

Benefits:
- Removes duplicate data
- Improves consistency
- Saves storage


------------------------------------------------------------


9. WHAT IS DENORMALIZATION?
Denormalization is the process of combining tables to improve performance.

Explanation:
Sometimes joins are expensive, so data is duplicated intentionally to make queries faster.

Example:
Storing user name directly in orders table.

Trade-off:
Faster reads but more redundancy.


------------------------------------------------------------


10. WHAT ARE CONSTRAINTS?
Constraints are rules applied to columns to maintain data integrity.

Types:

- NOT NULL → Cannot be empty
- UNIQUE → No duplicate values
- PRIMARY KEY → Unique + Not null
- FOREIGN KEY → Maintains relationship
- CHECK → Condition on values

Example:
Age must be greater than 18


------------------------------------------------------------


11. WHAT IS INDEXING?
Indexing is a technique to speed up data retrieval.

Explanation:
Index works like a book index, allowing fast search instead of scanning the whole table.

Example:
Searching user by ID becomes faster with index.

Trade-off:
Faster read, slower write.


------------------------------------------------------------


12. WHAT IS SQL?
SQL is a language used to interact with relational databases.

Explanation:
It is used to:
- Create tables
- Insert data
- Update data
- Delete data
- Query data

Example:
SELECT * FROM users;


------------------------------------------------------------


13. WHAT IS JOIN?
Join is used to combine data from multiple tables.

Types:

- INNER JOIN → Matching data only
- LEFT JOIN → All from left + matching from right
- RIGHT JOIN → All from right + matching from left

Example:
Users + Orders table joined using user_id


------------------------------------------------------------


14. WHAT IS SCHEMA IN RDBMS?
Schema defines the structure of tables in the database.

Explanation:
It includes:
- Table names
- Columns
- Data types
- Relationships


------------------------------------------------------------


15. WHAT IS DATA INTEGRITY?
Data integrity ensures accuracy and consistency of data.

Types:

- Entity Integrity → Primary key must be unique
- Referential Integrity → Foreign key must match
- Domain Integrity → Valid data types

Example:
Cannot insert order with invalid user_id
