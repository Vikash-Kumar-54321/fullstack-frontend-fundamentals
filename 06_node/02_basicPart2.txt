1. WHY ASYNCHRONOUS PROGRAMMING IS IMPORTANT
----------------------------------------------------------

Node.js uses asynchronous programming to handle many requests efficiently.

Explanation:
In synchronous systems, tasks execute one after another. If one task takes a long time, the entire system waits.

Example:

read file
query database
send response

If reading the file takes 5 seconds, the server cannot process other requests during that time.

Node.js solves this using asynchronous operations.


----------------------------------------------------------
2. CALL STACK
----------------------------------------------------------

The call stack is a data structure used to track function execution.

Explanation:
Whenever a function is called, it is pushed onto the call stack. When it finishes execution, it is removed.

Example:

function A()
function B()

Execution:

Call A → stack
Call B → stack
Return B
Return A


----------------------------------------------------------
3. SYNCHRONOUS EXECUTION
----------------------------------------------------------

In synchronous execution, tasks execute one by one.

Example:

console.log("Start")
console.log("Middle")
console.log("End")

Output:

Start
Middle
End

Each statement executes sequentially.


----------------------------------------------------------
4. ASYNCHRONOUS EXECUTION
----------------------------------------------------------

In asynchronous execution, long operations run in the background.

Example:

console.log("Start")

setTimeout(() => {
 console.log("Async Task")
}, 0)

console.log("End")

Output:

Start
End
Async Task

Explanation:
The async task runs after the main code finishes.


----------------------------------------------------------
5. WHAT IS THE EVENT LOOP
----------------------------------------------------------

The event loop is the mechanism that allows Node.js to perform non-blocking operations.

Definition:
It continuously checks the call stack and callback queue to decide what code to execute next.


Working Model:

Call Stack
↓
Node APIs
↓
Callback Queue
↓
Event Loop
↓
Call Stack


----------------------------------------------------------
6. CALLBACK QUEUE
----------------------------------------------------------

The callback queue stores callbacks from asynchronous operations.

Example:

setTimeout callback
HTTP request callback
File read callback

When the call stack becomes empty, the event loop moves callbacks from the queue to the stack.


----------------------------------------------------------
7. MICROTASK QUEUE
----------------------------------------------------------

The microtask queue stores higher priority asynchronous tasks.

Examples:

Promises (.then)
process.nextTick

Priority order:

1. Call Stack
2. Microtask Queue
3. Callback Queue


Example:

console.log("Start")

Promise.resolve().then(()=>{
 console.log("Promise")
})

console.log("End")

Output:

Start
End
Promise



----------------------------------------------------------
8. NODE.JS EVENT LOOP PHASES
----------------------------------------------------------

The event loop has multiple phases.

1. Timers
2. Pending callbacks
3. Idle/prepare
4. Poll
5. Check
6. Close callbacks


----------------------------------------------------------
9. TIMERS PHASE
----------------------------------------------------------

Executes callbacks scheduled by setTimeout and setInterval.

Example:

setTimeout(()=>{
 console.log("Timer executed")
},1000)


----------------------------------------------------------
10. POLL PHASE
----------------------------------------------------------

Handles incoming I/O events.

Examples:

File system operations
Database queries
Network requests


----------------------------------------------------------
11. CHECK PHASE
----------------------------------------------------------

Handles callbacks scheduled by setImmediate.

Example:

setImmediate(()=>{
 console.log("Immediate callback")
})


----------------------------------------------------------
12. CLOSE CALLBACK PHASE
----------------------------------------------------------

Handles closing events.

Example:

socket.on("close", ()=>{})


----------------------------------------------------------
13. setTimeout VS setImmediate
----------------------------------------------------------

Both schedule asynchronous tasks but in different phases.

setTimeout:
Executed in timers phase.

setImmediate:
Executed in check phase.

Example:

setTimeout(()=>console.log("timeout"),0)
setImmediate(()=>console.log("immediate"))

Execution order may vary depending on environment.


----------------------------------------------------------
14. process.nextTick
----------------------------------------------------------

process.nextTick executes before the event loop continues.

Example:

console.log("Start")

process.nextTick(()=>{
 console.log("Next Tick")
})

console.log("End")

Output:

Start
End
Next Tick


----------------------------------------------------------
15. PROMISE QUEUE
----------------------------------------------------------

Promises are handled in the microtask queue.

Example:

Promise.resolve().then(()=>{
 console.log("Promise executed")
})


----------------------------------------------------------
16. EXECUTION PRIORITY ORDER
----------------------------------------------------------

Priority order in Node.js:

1. Call Stack
2. process.nextTick
3. Promise Microtask Queue
4. Callback Queue


----------------------------------------------------------
17. COMPLETE EXECUTION EXAMPLE
----------------------------------------------------------

Code:

console.log("Start")

setTimeout(()=>{
 console.log("Timeout")
},0)

Promise.resolve().then(()=>{
 console.log("Promise")
})

console.log("End")


Execution:

Start
End
Promise
Timeout


Explanation:

Sync code executes first
Microtasks execute next
Timers execute last


----------------------------------------------------------
18. COMMON INTERVIEW QUESTIONS
----------------------------------------------------------

Q1: Why is Node.js fast?

Answer:
Because it uses non-blocking asynchronous I/O and event-driven architecture.


Q2: What is the event loop?

Answer:
The event loop manages asynchronous operations by moving callbacks from queues to the call stack.


Q3: Difference between setTimeout and setImmediate?

Answer:
They execute in different phases of the event loop.


Q4: What is the microtask queue?

Answer:
A high-priority queue for promises and process.nextTick.

----------------------------------------------------------

User sends API request

Request arrives

Database query executed asynchronously
