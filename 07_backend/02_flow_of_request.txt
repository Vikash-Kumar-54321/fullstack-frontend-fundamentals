INTRODUCTION
--------------------------------------------------------------------------------
Modern web applications rely on multiple technologies working together across
different layers of the internet. When a user types a URL in the browser and
presses Enter, many complex processes occur before the webpage finally appears
on the screen.

This document explains the entire flow in depth, starting from system
architecture concepts to the full networking and browser rendering process.

The main concepts covered include:

1. Software architecture models
2. Domain Name System (DNS)
3. TCP connection establishment
4. TLS handshake and secure communication
5. HTTP request and response
6. Backend server processing
7. Database queries
8. Browser rendering pipeline



================================================================================
SECTION 2: WHAT HAPPENS WHEN YOU TYPE A URL
================================================================================

Example URL:

https://www.example.com/products?id=10

URL stands for: Uniform Resource Locator

The browser parses the URL into components.

Protocol:
https

Domain:
www.example.com

Path:
/products

Query Parameters:
?id=10


================================================================================
SECTION 3: BROWSER CACHE CHECK
================================================================================

Before contacting the network, the browser checks if requested resources
already exist locally.

Examples of cached resources:

HTML files
CSS stylesheets
JavaScript files
Images
Fonts

If resources exist in cache and are still valid, the browser loads them
immediately without contacting the server.


================================================================================
SECTION 4: DNS LOOKUP
================================================================================

DNS stands for:

Domain Name System

Computers communicate using IP addresses, not domain names.

DNS converts domain names into IP addresses.

Example:

google.com → 142.250.183.206

DNS resolution process:

1. Browser cache check
2. Operating system cache
3. Router cache
4. ISP DNS server
5. Root DNS server
6. TLD server
7. Authoritative DNS server

Full forms:

ISP = Internet Service Provider
TLD = Top Level Domain

Examples of TLD:

.com
.org
.net

The authoritative DNS server returns the IP address of the website.
