=========================================================
BACKEND ARCHITECTURE FUNDAMENTALS
Monolithic vs Client-Server vs Microservices
=========================================================

1. INTRODUCTION TO SOFTWARE ARCHITECTURE
---------------------------------------------------------

Software Architecture refers to the overall structure and design
of a software system. It defines how different components of a
system are organized and how they communicate with each other.

Architecture determines:

- How frontend communicates with backend
- How backend processes requests
- How data is stored and accessed
- How scalable and maintainable the system is

In modern web applications, architecture is extremely important
because systems must handle thousands or even millions of users.

The three common architectures discussed are:

1. Monolithic Architecture
2. Client-Server Architecture
3. Microservices Architecture


=========================================================
2. MONOLITHIC ARCHITECTURE
=========================================================

Definition:
A Monolithic Architecture is a software design where the entire
application is built as a single unified codebase and deployed
as one application.

All modules of the system are tightly coupled and run together
in one server process.

Typical components inside a monolithic backend:

- Authentication
- User management
- Product management
- Payment handling
- Order processing
- Notification system

All these modules exist inside the same project.

Example Structure (Node.js Project)

project/
   routes/
   controllers/
   models/
   services/
   app.js

All functionalities run under one backend application.

Typical Flow

Frontend
   ↓
Backend Application (Single Codebase)
   ↓
Database

Example Technology Stack

Frontend: React / HTML / EJS
Backend: Node.js + Express
Database: MongoDB

This structure is still considered Monolithic because
all backend logic exists inside a single application.

Advantages of Monolithic Architecture

1. Simple to develop
All code exists in one place, making development easier.

2. Easy debugging
Since everything runs together, debugging becomes simpler.

3. Faster initial development
Startups can quickly build products without worrying about
service communication.

4. Easier deployment
Only one application needs to be deployed.

Disadvantages of Monolithic Architecture

1. Difficult to scale specific features
If one module becomes heavy (like payment system), the entire
application must be scaled.

2. Codebase becomes very large
Large monolithic systems can reach millions of lines of code.

3. Risky deployments
A small change requires redeploying the entire application.

4. Harder team collaboration
Large teams working on the same codebase can cause conflicts.

Despite disadvantages, monolithic architecture is ideal for:

- Small applications
- Early-stage startups
- Small development teams
